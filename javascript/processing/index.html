From the processors available in eunice-processors, the JavaScript NPM package runs the following automatically:
  1. Reverse file content. Requires option is-file-content-reversed.
  2. Filename "index" is removed from identifiers.
  3. Ensure there is a single root item with an identifier. Requires option root-item or package-names.
  4. Items that are files have "type: file" added to them.
  5. Items are ordered by their identifiers.
  6. Items are grouped by their identifiers (this is requires the previous ordering step).
  7. Anonymous items (i.e. those without an identifier) are given the identifier of their parent.
  8. Variables that depend on a single item of the same name are removed (e.g. some CommonJS requires or ES6 imports).
  9. Items are ordered so parameters, variables and files are last.
  10. Modify the stacks of all files, but not the sub-stacks within those files. Requires option modify-file-stacks-file.
  11. Independent items (i.e. those that don't depend on each other) are unstacked into the same level.
  12. The source directory is scanned for any files named .devsnicket-eunice-stack.yaml. Any structure defined inside these files is applied to any files also in the directory.
  13. If there is a single root item, that is anonymous and has only items and type, its items are flattened out into the root
