members

By default items are created for namespaces and types, but not members of those types.

Dependency counts between members of the same type aren't represented. Lists of dependencies between types don't specify the members involved.

To include members and have their stacks inferred:

eunice --members

project or solution

If the current directory has a single solution or project file it will be analysed by default. Alternatively a filename or path can be specified:

eunice Directory/MySolution.sln
